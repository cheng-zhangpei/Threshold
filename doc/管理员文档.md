## 管理员操作手册

本手册面向 Threshold 安全代理系统的管理员，涵盖系统初始化、设备管理、证书管理和日常运维操作。所有操作通过 `adminctl` 命令行工具完成。

### 前置条件

- Threshold Server 已编译并可运行
- adminctl 工具已编译（`go build -o adminctl ./cmd/adminctl/`）
- 所有命令在项目根目录下执行

### 一、系统初始化

系统首次部署时需要完成以下初始化步骤。每个步骤只需执行一次。

#### 1.1 启动 Server

```bash
go run ./cmd/server/ -config=./config/server.yaml
```

首次启动时，由于管理员账号尚未初始化，Server 会在控制台输出一次性口令：

```
========================================
  Admin not initialized.
  One-time passcode: a1b2c3d4e5f6789...
  Run: adminctl init -passcode <code> -user admin -pass <password>
========================================
```

请复制输出的 passcode，下一步需要使用。Server 保持运行，不要关闭。

#### 1.2 初始化管理员账号

```bash
go run ./cmd/adminctl/ init \
  -server 127.0.0.1:50051 \
  -passcode a1b2c3d4e5f6789... \
  -user admin \
  -pass your-secure-password
```

输出：

```
Admin initialized successfully.
Run 'adminctl login' to authenticate.
```

初始化成功后，passcode 文件自动删除，不可重复使用。后续如需重新初始化管理员，需要删除 bbolt 数据库中的 admin_credentials bucket 并重启 Server。

#### 1.3 初始化 CA（证书颁发机构）

```bash
go run ./cmd/adminctl/ ca init
```

输出：

```
CA initialized successfully.
  CA cert: ./data/ca/ca.crt
  CA key:  ./data/ca/ca.key
  Valid until: 2036-06-25
```

生成两个文件：
- `data/ca/ca.crt` — CA 公钥证书，需要分发到所有设备
- `data/ca/ca.key` — CA 私钥，仅管理员持有，绝不外泄

#### 1.4 签发服务端证书

```bash
go run ./cmd/adminctl/ cert issue \
  -type server \
  -hosts "127.0.0.1,localhost,10.0.0.1"
```

`-hosts` 参数填写 Threshold Server 所有对外暴露的 IP 地址和域名，多个地址用逗号分隔。客户端连接时会验证服务端证书的 SAN（Subject Alternative Name）中是否包含实际连接的地址。

输出：

```
Certificate issued:
  Type: server
  Cert: ./data/certs/server.crt
  Key:  ./data/certs/server.key
  Valid until: 2027-06-25
```

#### 1.5 签发 CLI 管理工具的客户端证书

```bash
go run ./cmd/adminctl/ cert issue \
  -type client \
  -uuid adminctl-client
```

输出：

```
Certificate issued:
  Type: client
  UUID: adminctl-client
  Cert: ./data/certs/adminctl-client.crt
  Key:  ./data/certs/adminctl-client.key
```

#### 1.6 重启 Server

Server 启动时会自动检测证书文件，检测到 `data/certs/server.crt` 和 `data/ca/ca.crt` 后启用 mTLS。重启 Server 使新签发的服务端证书生效：

```bash
go run ./cmd/server/ -config=./config/server.yaml
```

输出中应包含：

```
gRPC server: mTLS enabled
tcplistener: mTLS enabled (CA=./data/ca/ca.crt)
```

#### 1.7 验证 CLI 连接

```bash
go run ./cmd/adminctl/ login \
  -server 127.0.0.1:50051 \
  -user admin \
  -pass your-secure-password \
  -ttl 24h
```

输出：

```
Logged in as admin
Token expires: 2026-06-26T15:00:00+08:00
```

登录成功说明 mTLS 链路正常：adminctl 使用 `adminctl-client.crt` 向 Server 证明身份，Server 使用 `server.crt` 向 adminctl 证明身份。

### 二、管理员登录与 Token 管理

#### 2.1 登录

```bash
go run ./cmd/adminctl/ login \
  -server 127.0.0.1:50051 \
  -user admin \
  -pass your-secure-password \
  -ttl 24h
```

`-ttl` 参数指定 Token 有效期，支持 `1h`、`24h`、`168h`（7 天）等格式，最大不超过 7 天。Token 加密存储在 `data/admin.token` 文件中，权限 0600。

#### 2.2 查看 Token 状态

Token 保存在本地文件中，可以直接查看：

```bash
cat ./data/admin.token
```

Token 过期后所有管理操作会被拒绝，需要重新登录。

#### 2.3 登出

```bash
go run ./cmd/adminctl/ logout
```

输出：

```
Token removed. You have been logged out.
```

登出仅删除本地 Token 文件，服务端的 Token 会在过期后自动清理。

### 三、设备指纹管理

设备必须先注册指纹才能通过 Threshold Server 的安全校验。注册后设备的 UUID、操作系统类型和 IP 地址会被写入指纹树。

#### 3.1 登录（所有管理操作前置步骤）

```bash
go run ./cmd/adminctl/ login \
  -server 127.0.0.1:50051 \
  -user admin \
  -pass your-secure-password
```

#### 3.2 注册设备

```bash
go run ./cmd/adminctl/ register \
  -server 127.0.0.1:50051 \
  -uuid device-abc-123 \
  -os linux \
  -ip 10.0.0.50
```

输出：

```
Device registered: device-abc-123 (linux/10.0.0.50)
```

`-uuid` 为设备的唯一标识，`-os` 为操作系统类型（linux / windows / macos），`-ip` 为设备 IP 地址。如果不指定 `-ip`，CLI 会自动检测本机 IP。

IP 维度会影响指纹匹配精度。如果设备通过不同网络接口连接（如 localhost 和局域网 IP），需要将两种 IP 都注册，或在注册时省略 `-ip` 让 IP 维度自动跳过匹配。

#### 3.3 查看已注册设备

```bash
go run ./cmd/adminctl/ list -server 127.0.0.1:50051
```

输出：

```
UUID                                     OS         IP
---------------------------------------- ---------- ---------------
device-abc-123                           linux      10.0.0.50
fallback-uuid-12345678                   linux      192.168.31.24

Total: 2 device(s)
```

#### 3.4 注销设备

```bash
go run ./cmd/adminctl/ unregister \
  -server 127.0.0.1:50051 \
  -uuid device-abc-123 \
  -os linux \
  -ip 10.0.0.50
```

输出：

```
Device unregistered: device-abc-123
```

注销后该设备的连接请求会被指纹匹配引擎拒绝，写入告警队列。

### 四、证书管理

证书是 mTLS 双向认证的载体。每个需要连接 Threshold Server 的设备都需要一张由 CA 签发的客户端证书。

#### 4.1 签发 Mode 1/2 客户端证书

Mode 1/2 客户端（Go 程序）的证书签发后，需要将证书文件分发到客户端设备。

```bash
go run ./cmd/adminctl/ cert issue \
  -type client \
  -uuid device-abc-123
```

输出：

```
Certificate issued:
  Type: client
  UUID: device-abc-123
  Cert: ./data/certs/device-abc-123.crt
  Key:  ./data/certs/device-abc-123.key
  Valid until: 2027-06-25
```

分发到目标设备：

```bash
scp ./data/ca/ca.crt user@10.0.0.50:/opt/threshold/data/ca/
scp ./data/certs/device-abc-123.crt user@10.0.0.50:/opt/threshold/data/certs/
scp ./data/certs/device-abc-123.key user@10.0.0.50:/opt/threshold/data/certs/
```

客户端设备上需要放置三个文件：
- `ca.crt` — 验证服务端证书
- `device-abc-123.crt` — 设备自己的证书
- `device-abc-123.key` — 设备自己的私钥

客户端启动时会自动检测 `data/certs/<device-uuid>.crt` 是否存在，存在则启用 mTLS，不存在则降级为无 TLS 连接。

#### 4.2 签发 Mode 3 设备证书

Mode 3 设备（LD_PRELOAD）的证书签发后，需要将证书文件分发到目标设备并设置环境变量。

```bash
go run ./cmd/adminctl/ cert issue \
  -type client \
  -uuid lab-server-001
```

分发到目标设备：

```bash
scp ./data/ca/ca.crt user@10.0.0.100:/opt/threshold/certs/
scp ./data/certs/lab-server-001.crt user@10.0.0.100:/opt/threshold/certs/
scp ./data/certs/lab-server-001.key user@10.0.0.100:/opt/threshold/certs/
```

目标设备上设置环境变量：

```bash
export THRESHOLD_CA_CERT=/opt/threshold/certs/ca.crt
export THRESHOLD_CLIENT_CERT=/opt/threshold/certs/lab-server-001.crt
export THRESHOLD_CLIENT_KEY=/opt/threshold/certs/lab-server-001.key
export THRESHOLD_DEVICE_UUID=lab-server-001
```

然后通过 LD_PRELOAD 运行目标程序：

```bash
LD_PRELOAD=/opt/threshold/libthreshold/libthreshold.so \
  curl http://10.0.0.200/api/data
```

如果未设置环境变量，`.so` 会降级为单向 TLS（数据仍然加密，但不提供客户端证书）。如果服务端要求 mTLS，降级的连接会被拒绝。

#### 4.3 自定义证书有效期

默认有效期 365 天，可通过 `-days` 参数指定：

```bash
go run ./cmd/adminctl/ cert issue \
  -type client \
  -uuid device-abc-123 \
  -days 730
```

#### 4.4 证书续期

证书过期前，重新签发同 UUID 的证书即可：

```bash
go run ./cmd/adminctl/ cert issue \
  -type client \
  -uuid device-abc-123
```

新证书会覆盖旧文件，将新证书分发到设备并重启应用即可。

### 五、设备吊销

当设备丢失、被入侵或不再需要访问时，需要将其从系统中移除。

#### 5.1 从指纹树注销

```bash
go run ./cmd/adminctl/ unregister \
  -server 127.0.0.1:50051 \
  -uuid device-abc-123 \
  -os linux \
  -ip 10.0.0.50
```

注销后设备的连接请求会在指纹匹配阶段被 BLOCKED，即使设备仍持有有效的 TLS 证书也无法访问业务后端。

#### 5.2 收回证书文件

从设备上删除证书文件：

```bash
ssh user@10.0.0.50 "rm -f /opt/threshold/data/certs/device-abc-123.*"
```

删除后设备无法建立 mTLS 连接，在 TLS 握手阶段就会被拒绝。

#### 5.3 双重保障

吊销后的安全状态：

```
TLS 层：证书文件被删除 → TLS 握手失败 → 连接在网络层被拒绝
指纹层：UUID 从指纹树删除 → 即使 TLS 通过，应用层也会 BLOCKED
```

两层独立防护，任意一层的吊销生效都能阻止设备访问。

### 六、日常运维

#### 6.1 查看 Server 日志

Server 日志同时输出到控制台和 `log/server.log` 文件：

```bash
tail -f log/server.log
```

关键日志标识：

```
[ADMIN] admin initialized          — 管理员初始化
[ADMIN] login success              — 管理员登录成功
[ADMIN] device registered          — 设备注册
[ADMIN] device unregistered设备注销
tcplistener: mTLS enabled          — Mode 3 mTLS 启用
FINGERPRINT REJECTED               — 指纹匹配失败
tcplistener: session end           — 连接结束
```

#### 6.2 查看已注册设备

```bash
go run ./cmd/adminctl/ login -server 127.0.0.1:50051 -user admin -pass xxx
go run ./cmd/adminctl/ list -server 127.0.0.1:50051 -limit 100
```

#### 6.3 查看已签发证书

```bash
ls -la data/certs/
```

`.crt` 文件是证书，`.key` 文件是私钥。文件名即设备 UUID。
